# Al-Hafeez Smart Notebook

A premium cafe sales tracker and menu management app built with React + Vite + Firebase. Track daily sales, manage your menu, review history, and see revenue summaries.

## Run & Operate

- `pnpm --filter @workspace/cafe-tracker run dev` — run the cafe tracker (port 18770)
- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React 19 + Vite 7 + Tailwind CSS v4
- Routing: wouter
- Animations: framer-motion
- Database: Firebase Firestore (frontend-only, no backend)
- Icons: lucide-react
- UI: shadcn/ui components

## Where things live

- `artifacts/cafe-tracker/` — main app
- `artifacts/cafe-tracker/src/lib/firebase.ts` — Firebase initialization
- `artifacts/cafe-tracker/src/lib/firestore.ts` — Firestore CRUD helpers
- `artifacts/cafe-tracker/src/pages/` — SalesEntry, History, Summary, Menu
- `artifacts/cafe-tracker/src/components/BottomNav.tsx` — bottom navigation
- `artifacts/cafe-tracker/vercel.json` — Vercel SPA rewrite rules

## Architecture decisions

- Pure frontend + Firebase architecture: no Express server needed for the cafe tracker
- Firestore real-time listeners (onSnapshot) for live data across pages
- Seeds default menu items to Firestore on first load if collection is empty
- Dark-only theme (adds `dark` class to html element in main.tsx)
- Vercel SPA rewrites in vercel.json ensure all routes work on direct load/refresh

## Product

- **Sales Entry (/)**: Tap menu items to add to cart, pick payment type, save with one tap. Custom date picker for recording old sales.
- **History (/history)**: Notebook-style sales grouped by date. Edit and delete entries. Change payment type, quantities, and date.
- **Summary (/summary)**: Revenue breakdown by period (today/week/month) and payment type (Cash, UPI, Split, Pending). Top items list.
- **Menu (/menu)**: Full CRUD for menu items. Images, categories, prices, enable/disable toggle.

## User preferences

- Black + Gold premium theme (always dark)
- Currency: Indian Rupees (₹)
- Payment types: Cash, UPI, Split, Pending
- Mobile-first (max-width 430px)
- No emojis in UI

## Gotchas

- Firebase env vars must be set with VITE_ prefix for frontend access
- Run codegen (`pnpm --filter @workspace/api-spec run codegen`) after OpenAPI spec changes
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only, for API server)
- Required API server env: `DATABASE_URL` — Postgres connection string

## Vercel Deployment

- `vercel.json` is at `artifacts/cafe-tracker/vercel.json`
- Set all VITE_FIREBASE_* environment variables in Vercel dashboard
- Output directory: `dist/public`
- Build command: `vite build --config vite.config.ts`
